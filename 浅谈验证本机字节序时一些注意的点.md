### 浅谈验证本机字节序时一些注意的点



Q: 

```cpp
#include <iostream>

using namespace std;

bool is_big_endian() {
	int num = 1;
	if (*(char*)(&num) == 1) {
		return false;//小端序
	}
	else {
		return true;//大端序
	}
}

int main() {
	int num = 0x12345678;
	char* p = (char*)&num;

	if (is_big_endian()) {
		cout << "当前系统为大端序" << endl;
		cout << "字节序为：";
		for (int i = 0; i < sizeof(num); i++) {
			cout << hex << (int)*(p + i) << " ";
		}
		cout << endl;
	}
	else {
		cout << "当前系统为小端序" << endl;
		cout << "字节序为：";
		for (int i = sizeof(num) - 1; i >= 0; i--) {
			cout << hex << (int)*(p + i) << " ";
		}
		cout << endl;
	}

	return 0;
}
```

num的地址转换为char*类型，解引用时因为char只有1字节，所以解引用的值为num(int类型)第一个字节处的数据 我的理解对吗 




A:

**你的理解完全正确！**

你准确地抓住了这段代码的核心原理。为了让你更清晰，我们可以把这句话拆解成三个步骤来详细剖析：

### 1. `&num` 获取起始地址
`num` 是一个 `int` 类型，占用 4 个字节。`&num` 获取的是这 4 个字节中**首字节（最低地址）**的内存地址。此时它的类型是 `int*`。

### 2. `(char*)` 改变指针的“视角”和“步长”
将 `int*` 强转为 `char*`，内存地址本身的值并没有变，但编译器看待这块内存的方式变了：
*   **步长变了**：`int* + 1` 会向后偏移 4 个字节；`char* + 1` 只会向后偏移 1 个字节。
*   **解引用范围变了**：解引用 `int*` 时，编译器会一次性读取 4 个字节拼成 `int`；而解引用 `char*` 时，编译器**只会读取 1 个字节**。

### 3. `*` 解引用取出第一个字节
因为 `char` 只占 1 个字节，所以 `*(char*)(&num)` 就是**从 `num` 的起始地址处，读取 1 个字节的数据**。

---

### 结合字节序（Endianness）的理解

这正是 `is_big_endian()` 函数能够判断系统字节序的原理：

*   `int num = 1;` 在内存中的二进制是 `...00000001`。
*   在十六进制中，它的最低位字节是 `0x01`，高位字节是 `0x00`。

**如果是小端序（Little-Endian）：**
*   低位数据放在低地址处。
*   内存排列（从低地址到高地址）：`01 00 00 00`
*   `*(char*)(&num)` 读取最低地址处的 1 个字节，得到 `0x01`（即等于1），所以返回 `false`。

**如果是大端序（Big-Endian）：**
*   高位数据放在低地址处。
*   内存排列（从低地址到高地址）：`00 00 00 01`
*   `*(char*)(&num)` 读取最低地址处的 1 个字节，得到 `0x00`（即等于0），所以返回 `true`。

---

### 💡 一个代码优化建议（关于打印隐患）

虽然你的理解是对的，但你的代码在打印字节时存在一个**潜在的隐患**，建议将 `char*` 改为 `unsigned char*`。

**原因**：
在你的代码中 `char* p = (char*)&num;`，`char` 在很多系统上默认是**有符号的**。
当某个字节的值大于 `127`（例如 `0x8A`），它会被视为负数。当你执行 `(int)*(p + i)` 时，会发生**符号扩展**，导致输出错误。

例如：如果某个字节是 `0x8A`，强转为 `int` 后会变成 `0xFFFFFF8A`，打印出来就会是一长串 `ffffff8a`，而不是期望的 `8a`。

**修改后的 `main` 函数：**

```cpp
int main() {
	int num = 0x12345678;
	unsigned char* p = (unsigned char*)&num; // 改为 unsigned char*

	if (is_big_endian()) {
		cout << "当前系统为大端序" << endl;
		cout << "字节序为：";
		for (int i = 0; i < sizeof(num); i++) {
			cout << hex << (int)*(p + i) << " "; // 现在即使大于0x7F也不会符号扩展
		}
		cout << endl;
	}
	else {
		cout << "当前系统为小端序" << endl;
		cout << "字节序为：";
		for (int i = sizeof(num) - 1; i >= 0; i--) {
			cout << hex << (int)*(p + i) << " ";
		}
		cout << endl;
	}

	return 0;
}
```

你可以尝试把 `num` 的值改为 `0x80ABCDEF`，用 `char*` 和 `unsigned char*` 分别运行一下，就能明显看到区别了！
